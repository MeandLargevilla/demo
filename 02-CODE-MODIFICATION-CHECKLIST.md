# Code Modification Checklist

## Before Modifying Code:
- [ ] Ensure you have the latest version of the codebase.
- [ ] Review the current code to understand its functionality.
- [ ] Identify related components that may be affected by the changes.

## During Modification:
- [ ] Write clean and maintainable code.
- [ ] Make sure to comment on complex logic.
- [ ] Update documentation where necessary.

## After Modification:
- [ ] Test the changes thoroughly.
- [ ] Review your code for any unused imports or variables.
- [ ] Ensure that existing functionality is not broken.

## Pull Request Checklist:
- [ ] Include a clear and concise description of the changes.
- [ ] Reference any related issues.
- [ ] Label the pull request appropriately.
- [ ] Request a review from relevant team members.

## Misc:
- [ ] Ensure code adheres to project style guides.
